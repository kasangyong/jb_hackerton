## 🤖 실제 AI 백엔드 — *데모의 자산 케어 챗을 진짜 AI로* (본선)

데모의 **② 자산 케어** 챗은 정적 답변이 아니라, 로컬에서 **실제 EXAONE-3.5 RAG**가 답하도록 구현했다. 서버가 꺼져 있으면 예시 답변으로 자동 폴백한다(발표장 안전장치).

```
[데모 챗 UI]  ── POST /chat ──▶  [FastAPI :8000]  ── 검색 ──▶  docs/*.md (공식 출처 6종)
                                              └── 생성 ──▶  Ollama (EXAONE-3.5-2.4B)
```

- **검색**: BM25 + BGE-M3(임베딩) → **RRF** 융합 (하이브리드, 섹션 맥락 결합)
- **생성**: EXAONE-3.5-2.4B (Ollama, **로컬·무료·CPU 구동**)
- **가드레일**: 근거 유사도 미달 시 LLM 호출 없이 **"상담사 연결"** → 환각 차단
- **출처**: 6개 문서 전부 **실제 공식 기관 원문 기반 + 인용 URL** (한국주택금융공사·보건복지부·예금보험공사·금융위원회·은행연합회·국민건강보험공단)

**실행** ([`backend/README.md`](backend/README.md) 참고):
```bash
ollama pull exaone3.5:2.4b && ollama pull bge-m3   # 한 번만 (~3GB)
cd backend && pip install -r requirements.txt
python server.py                                    # → http://127.0.0.1:8000
```

**검증됨** (CPU 노트북 실측): 콜드스타트 첫 답변 ~50초, 이후 **답변당 1~2초**.
- "예금자보호 한도?" → *"2025.9.1부터 원금+이자 1억원"*
- "기초연금 얼마 받아요?" → *"단독 342,510원 / 부부 각 274,000원"*
- "비트코인 사도 돼요?"(범위 밖) → 가드레일 작동 → *"상담사에게 연결"*

## 1. 문제 정의 — *지킬 자산을 잃으면, 관리할 자산도 없다*

남들은 자산을 **'불리는'** WM(로보어드바이저)을 말하지만, JB의 핵심 고객인 **지방 고령층의 진짜 문제는 '가진 자산을 잃는 것'** 이다.

| 지표 | 수치 | 의미 |
|---|---|---|
| 전남 고령화율 | **30.6%** (전국 1위) | 전북 25.4% · 2052년에도 전남 1위 전망 |
| 전북 은행 점포 | **−23.4%** | 149개(2011) → 114개(2023) — 창구 의존 고령층 소외 |
| 고령층 디지털 역량 | **55.3%** | 스마트폰은 있으나 '쓸 줄'을 모름 (NIA 2024) |
| 60대+ 보이스피싱 피해 | **10년새 5배 ↑** | 전 연령 피해 비중 1위(30.6%), 2025 피해 1조원 돌파 전망 |

> 한 번의 사기로 평생 모은 노후자금이 사라지면 관리할 자산 자체가 없다. **대형은행이 떠난 이 자리를 지킬 곳은 지방은행 JB뿐.**

## 2. 솔루션 개요 — 대화와 거래를 함께 지켜보는 선제적 에이전트

```
[입력]  ① 음성·텍스트 대화        ② 거래 이벤트(이체·출금)   ← 은행만 가진 데이터
            │                          │
            ▼                          ▼
       [의도 라우터]              [거래 모니터(24h 상시)]
            │                          │
            └──────────┬───────────────┘
                       ▼
        [AI Agent 오케스트레이터]  plan → tool → self-check(critic) → 응답 / 선제 트리거
        │            │              │              │
        ▼            ▼              ▼              ▼
  ★거래단 사기방어  자산케어 RAG   쉬운말·음성     선제 알림·연계
  (룰+개인화         (BM25+BGE-M3   (LLM + TTS)     (상담사/가족)
   베이스라인        +RRF+Reranker)
   →신호등🟢🟡🔴)
                       │
                       ▼
              [가드레일] 근거 없으면 "사람 연결" / 사기 의심 시 "송금 멈춤"을 답변보다 우선
                       │
                       ▼
       [출력] 음성 안내 · 큰 글씨/고대비 카드 · 원터치 액션  +  JB 본부 상황판(관리자 뷰)
```

**왜 'Agent'인가** — 단순 Q&A 챗봇이 아니라 ① **선제성**(거래를 스스로 지켜보다 먼저 개입) ② **도구 사용**(RAG·FDS·알림·핸드오프) ③ **자기검증**(가드레일) 구조를 갖춘다.

## 4. 히어로 — 「송금 직전 30초」와 개인화 베이스라인

**같은 300만원이라도 결과가 다르다.** 평소 아들에게 보내던 박 어르신은 통과(🟢), 검찰 사칭에 속아 낯선 계좌로 보내려는 김 어르신은 차단(🔴). **금액이 아니라 '이 어르신다움'으로 판단**한다.

**2층 방어 엔진** (데모에서 실제로 계산됨):

```js
// 1층 — 2024 FDS 가이드라인 룰 (설명가능)
if (age >= 65 && isNew && amount >= 1_000_000) flags.push(["신규 수취인 고액 이체", 50]);
if (hour < 6)                                  flags.push(["심야 거래", 20]);

// 2층 — 개인화 베이스라인 (이 사용자의 '평소' 대비 이탈)
if (!known)        flags.push(["평생 처음 보는 수취인", 25]);
const z = (amount - mean(baseAmounts)) / std(baseAmounts);
if (z >= 3)        flags.push([`평소(${mean})보다 비정상적으로 큰 금액 (z=${z})`, 35]);
if (!usualHours.includes(hour)) flags.push(["평소 거래하지 않던 시간대", 15]);

// 가중 합산 → 위험 신호등
score >= 70 ? "위험 🔴 (송금 멈춤)" : score >= 35 ? "주의 🟡" : "안전 🟢"
```

> 룰만 쓰면 "if문"으로 평가절하된다. **사용자별 평소 패턴(베이스라인) 대비 이탈**을 z-score로 잡아 *"진짜 FDS의 사고방식"* 을 구현했다. (확장: Isolation Forest → GNN 사기링 탐지)
