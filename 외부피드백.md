# 외부 AI(GPT·Gemini) 공격적 피드백 받는 법

> 목적: 본선 전에 **GPT·Gemini를 '떨어뜨리려는 깐깐한 심사위원'으로 세워** 약점을 미리 다 털어내기.
> 핵심: *칭찬 금지, 공격만.* 그래야 진짜 구멍이 보인다.

---

## 1. 무엇을 주나 (파일 세트)

GPT·Gemini는 맥락 한계가 있으니, **이 4개 핵심 문서**만 붙여넣거나 GitHub 링크를 준다. (이 4개면 전체 그림이 다 담김)

| 파일 | 무엇 |
|---|---|
| `README.md` | 전체 개요·기능·ROI |
| `발표설계.md` | 배경·문제·작동원리(실제 기술)·PPT 뼈대 |
| `기술상세.md` | 우리가 짠 것의 내부 동작 |
| `사용자시나리오.md` | 무설치 앰비언트·송금 직전 30초 전달 흐름 |

- **방법 A (추천)**: GitHub 링크 `https://github.com/kasangyong/jb_hackerton` 를 주고 "이 레포의 README·발표설계·기술상세·사용자시나리오.md를 읽고" 라고 지시 (브라우징 되는 모델).
- **방법 B**: 위 4개 md 내용을 그대로 붙여넣기.
- (선택) 데모 화면 캡처 2~3장(① 사기 방어, 🎬 실감 시나리오, 🏦 본부 상황판) 첨부.

## 2. 그대로 복붙하는 공격 프롬프트

```
너는 JB금융그룹 Fin:AI Challenge 본선 심사위원 중 가장 깐깐하고 냉소적인 사람이다.
너의 목표는 이 팀(서비스명 '평생곁에', 지정주제 1 개인 라이프케어 AI)을 떨어뜨리는 것이다.
칭찬은 한 줄도 하지 마라. 오직 약점, 허점, 비현실적인 주장만 공격해라.

[첨부/링크한 문서를 근거로] 아래를 표로 작성해라:

1) 치명적 약점 TOP 10 — 각 항목마다:
   - 약점 (구체적으로, 문서의 어느 주장이 문제인지 인용)
   - 왜 치명적인가 (심사 감점 포인트)
   - 내가 심사 때 던질 '킬러 질문' (한 문장)
   - 이 팀이 막으려면 어떤 답/근거가 필요한가

2) "그냥 ~아니냐" 공격 5개 — 예: "결국 if문 + z-score 아니냐", "RAG는 그냥 챗봇 아니냐" 식으로,
   기술을 평가절하하는 가장 아픈 프레이밍.

3) 비현실성 공격 — 실제 은행 도입 시 막히는 지점(규제·데이터·고령자 사용성·비용)을 최대한 가혹하게.

4) 경쟁·차별성 공격 — 통신사/카드사 FDS, 기존 은행 앱 대비 "이게 왜 특별하냐".

5) 마지막에: 이 팀이 1등을 못 할 가장 큰 이유 1가지.

말투는 비판적이고 직설적으로. 듣기 좋은 말 금지.
```

## 3. 추가로 꼭 캐물을 '아픈' 질문들 (직접 던져도 됨)

문서를 줬는데 모델이 무뎌지면, 아래를 *콕 집어* 다시 물어라:

- **개인화**: "고령 고객은 거래가 월 몇 건인데, 그 적은 데이터로 개인화 베이스라인이 통계적으로 의미가 있나? 또래평균 보완한다는데 그럼 '개인화'가 맞나?"
- **모델**: "EXAONE 2.4B는 작은 모델이라 금융 답변에서 틀릴 텐데, RAG·가드레일로 막는다지만 근거 검색이 틀리면? 환각률을 측정한 숫자가 있나?"
- **데이터**: "사기계좌 블랙리스트는 무료 공개데이터에 계좌번호가 없어 샘플로 시연했다는데, 그럼 핵심 기능이 데모용 가짜 아니냐?"
- **전화/채널**: "ARS 자동전화가 FDS 가이드라인 권고라지만, 그건 *은행 인프라*다. 너희가 만든 건 뭐냐? 결국 기획서 아니냐?"
- **주제 적합성**: "주제는 'LifeLong WM(자산 통합관리)'인데 너희는 사기방어만 한다. 자산·연금·부동산 통합 분석은 어디 있나?"
- **ROI**: "530만원·12,840명 숫자의 출처와 가정은? 12,840명은 '가정치'라며, 그럼 ROI가 허구 아니냐?"
- **솔로**: "1인 프로젝트인데 실제 은행 연동·운영을 감당할 수 있나? 완성도가 팀 대비 떨어지지 않나?"

## 4. 피드백을 받은 뒤 (활용법)

1. 나온 약점을 **'이미 답 있음 / 보강 필요 / 인정하고 한계로 명시'** 3분류.
2. '보강 필요'만 골라 **발표 Q&A 탄창**에 반영 (각 약점당 30초 답변 준비).
3. 같은 프롬프트를 **GPT와 Gemini 둘 다**에 돌려 교집합(공통 지적)부터 처리 — 그게 진짜 약점.
4. 고친 뒤 **2차로 다시** 공격시켜 약점이 줄었는지 확인.

> 팁: "이 답변이면 심사위원을 납득시킬 수 있겠냐? 다시 공격해봐"로 **반복 압박**하면 답변이 단단해진다.
