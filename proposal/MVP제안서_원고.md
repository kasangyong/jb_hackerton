## [슬라이드 2] 문제 정의 (Problem Definition)

**관점 전환**: 다른 모든 서비스는 자산을 *불리는* WM(로보어드바이저)을 말한다. 그러나 **JB의 핵심 고객인 지방 고령층에게 진짜 문제는 '가진 자산을 잃는 것'** 이다.

**누가**: JB 핵심 영업권(전북·전남) 거주 **만 65세 이상 고령 고객**

**무슨 문제**
1. **빠른 초고령화**: 전남 고령화율 **30.6%(전국 1위)**, 전북 25.4%. 2052년에도 전남 1위 전망(통계청).
2. **사라지는 점포**: 전북 일반은행 점포 2011년 149개 → 2023년 114개(**-23.4%**). 은행 이동거리 20km 초과 상위 30곳 중 26곳이 초고령 지역.
3. **못 넘는 디지털 장벽**: 고령층 디지털 역량 **55.3%**(NIA 2024) — 스마트폰은 있으나 '쓸 줄'을 모름.
4. **노후자금을 노리는 표적 사기**: 60대+ 보이스피싱 피해 비중 **30.6%(전 연령 1위)**, 60대 피해 10년새 **5배**, 연 피해액 **1조원 돌파 전망**. 기관사칭형 51%.

**왜 중요한가**: 한 번의 사기로 **평생 모은 노후자금이 사라지면, 관리할 자산 자체가 없다.** 대형은행이 떠난 이 자리를 책임질 곳은 지방은행 JB뿐. 2024 FDS 가이드라인은 부실 시 은행에 피해 분담을 요구 — **방치는 곧 비용**.

**KPI**: 고령 고객 사기 **사전 차단율↑·피해액↓** / 단순 문의 셀프서비스 전환↑ / 고령 채널 유지율↑

## [슬라이드 3] 제안 솔루션 개요 (Solution Overview)

**한 문장**: 어르신의 **대화**와 **거래**를 함께 지켜보며, 위험한 송금 **직전에 먼저** 개입해 노후자금을 지키는 **선제적 자산 방어 에이전트**.

**전체 구조**
```
[입력] ① 음성/텍스트 대화   ② 거래 이벤트(이체·출금)  ← 은행만 가진 데이터
   │
[에이전트 코어] 의도 파악 → 도구 호출 → 자기검증 → 응답 / (선제) 위험하면 먼저 멈춰 세운다
   │
   ├─ ★ 거래단 사기 방어 (히어로) : FDS 룰 + 개인화 베이스라인 → 위험 신호등 🟢🟡🔴
   ├─ 생애주기 자산 케어 (조연) : RAG로 연금·예금·제도 근거 인용 안내
   ├─ 쉬운말 금융비서 : 전문용어 → 쉬운 말
   └─ 선제 알림 & 사람 연결 : 이상거래·만기 먼저 알림, 상담사 핸드오프
   │
[가드레일] 근거 없으면 "상담사 연결" / 사기 의심 시 "송금 멈춤"을 답변보다 우선
   │
[출력] 음성 + 큰 글씨·고대비 카드 + 원터치 액션
```

## [슬라이드 4] 주요 기능 정의 (Key Features)

> **핵심: 기능을 나열하지 않는다. 히어로 하나로 각인시킨다.**

| 기능 | 역할 | 해결 | MVP |
|---|---|---|---|
| **F2. 거래단 사기 방어 — 「송금 직전 30초」** | ★ **히어로** | 송금 버튼 직전, 은행이 먼저 개입해 노후자금 방어 | ✅ 핵심 |
| F1. 생애주기 자산 케어 | 조연 | 연금·만기·의료비·상속을 쉬운말+근거로 안내 | ✅ 핵심 |
| F3. 쉬운말 금융비서 | 보조 | "이번 달 왜 돈이 부족해?" 자연어 해설 | △ 부분 |
| F4. 선제 알림 & 사람 연결 | 보조 | 이상거래·만기 먼저 알림, 상담사/가족 연계 | ○ 설계 |

**F2를 'if문'이 아니게 만드는 깊이 (= 차별화)**:
- **1층 룰**(2024 FDS 가이드라인): 고령·신규수취인·고액·심야

## [슬라이드 5] 데이터 및 기술 활용 (Data & Tech)

**활용 데이터 (전부 공개 / 합성)**
- 금융상품·제도: 주택연금·기초연금 안내, 예적금·보증·대출 상품설명서
- 법령·가이드라인: 금융소비자보호법, 고령금융소비자 보호기준, **2024 FDS 가이드라인**
- 사기 패턴: 경찰청·금감원 보이스피싱 통계·유형, KorCCVi 공개 데이터셋
- 거래: **합성 데이터(개인정보 0)** — 사용자별 베이스라인 구성용

**처리 / 모델 / 도구**
- **거래 사기 방어(F2)**: 설명가능 룰 + **개인화 베이스라인 이상탐지(z-score·신규 수취인·시간대 이탈)** → 가중 위험 스코어러 → 신호등. *확장*: Isolation Forest → GNN.
- **자산 케어(F1)**: 하이브리드 RAG — BM25(kiwipiepy)+Dense(BGE-M3, Qdrant)+**RRF**+Cross-Encoder Reranker
- **LLM**: EXAONE-3.5(로컬) / **HyperCLOVA X SEED**(온프레미스·한국어, JB-네이버클라우드 제휴 정합)
- **에이전트**: 경량 상태머신(plan→tool→critic) + 선제 트리거 / **UI**: Streamlit

**기술적 제약 & 해결**
- *할루시네이션(금융=치명적)* → 근거 인용 강제 + 가드레일 + "모르면 사람 연결"
- *개인정보 보안* → 온프레미스 sLLM, 합성 데이터, 최소 수집
- *고령 사용성* → 음성 우선·한 화면 한 결정·반복 친화 (HCI 연구 근거)

## [슬라이드 6] 사용자 시나리오 / 유즈케이스 (User Scenario)

**주요 사용자**: 전북·전남 거주 70대 김OO 어르신 (스마트폰 보유, 디지털 자신감 낮음)

**★ 히어로 시나리오 — 「송금 직전 30초」 (F2)**
1. 김 어르신, "검찰"이라는 전화에 속아 모바일뱅킹으로 **300만원**을 처음 보는 계좌에 이체 시도
2. 평생곁에가 거래 이벤트 감지 → **1층 룰**(고령·신규수취인·고액) + **2층 베이스라인**(평소 30만원·자녀 계좌와 완전히 다름) → **🔴 위험**
3. 송금 화면 위에 **먼저** 개입: *"잠깐만요. 평소와 너무 다른 거래예요. 이건 보이스피싱이 의심돼요. 지금 멈추고 가족이나 112에 확인하세요."*
4. 원터치 [상담사 연결] / [가족에게 알림]
> **대비 컷(데모 킬러)**: 같은 300만원이라도, *평소 자녀에게 정기 송금하던 박 어르신*은 그대로 통과(🟢). → "개인화 방어"를 한눈에 증명.

**조연 시나리오 — 쉬운말 자산케어 (F1)**
- 어르신: *"주택연금 받으면 기초연금도 같이 받을 수 있어?"* → RAG 근거 인용 → *"네, 같이 받을 수 있어요. 다만 금액에 따라 조금 달라질 수 있어요. (근거: 주택연금 안내)"*

## [슬라이드 7] 기대 효과 및 향후 확장성 (Expected Impact)

**정량 효과 (가설 KPI)**
- 고령 고객 사기 **사전 차단율↑**, 피해액·은행 분담 리스크↓ (직접 비용 절감)
- 단순 문의 셀프서비스 전환↑ → 점포 축소 환경의 서비스 공백 보전
- 고령 디지털 채널 유지율↑ (고객 이탈 방지)

**정성 효과**
- **지역 동반자 브랜드**: 전북·전남 초고령 사회의 "평생 자산 지킴이"
- **포용금융·ESG**: 사회적 가치 + 금융당국·지자체 정합 (JB 공동전산센터·지역 밀착과 결)
- **직원 부담 경감**: 단순·반복 응대 → AI / 사람은 고난도·정서 케어

**향후 확장성**
- 단계: ① 개인 고령 자산 방어 → ② 가족 연계(자녀 알림·동의 보호) → ③ 소상공인 사장님 자금 케어 → ④ 전 연령 생애주기 케어
- 기술 고도화: 베이스라인 → **GNN 이상탐지**, sLLM **도메인 파인튜닝**, 음성(안티딥보이스 연계), **HyperCLOVA X** 기반 그룹 온프레미스 배포
- **기술 일반화 제언**: 우리의 3대 엔진(**개인화 이상탐지 · 근거 RAG+가드레일 · 온프레미스 sLLM**)은 사기방어 전용이 아니라 *"이 주체답지 않은 행동을 근거와 함께 잡는"* 범용 엔진. → **자본시장 불공정거래 감시(시세조종·미공개정보), AML(의심거래보고), 내부통제·준법 자문, 여신 사기**로 그대로 확장 (JB자산운용·인베스트먼트·우리캐피탈 등 전 계열 연계)
