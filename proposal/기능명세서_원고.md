## 1. 서비스 개요 (Service)

| 항목 | 내용 |
|---|---|
| **서비스명** | 평생곁에 — *평생 곁에서 노후자금을 지킵니다* |
| **한줄 소개** | 고령자의 자산관리는 '불리기'가 아니라 '지키기'다. 위험한 송금 직전 30초에 은행이 먼저 개입해 노후자금을 지키는 선제적 자산 방어 AI 에이전트 |
| **개발 목표** | 초고령·디지털소외·표적사기에 노출된 전북·전남 고령 고객의 노후자금을, 은행만 가진 거래 데이터로 ① 거래단 사기 방어(히어로) ② 쉬운 자산 케어로 '지킨다'. WM을 자산 증식이 아닌 **자산 방어(Wealth Defense)**로 재정의 |
| **타겟 사용자** | 만 65세 이상 지방(전북·전남) 거주 고령 금융소비자 (1차) / 가족·창구 직원 (2차) |
| **기대 효과** | 사기 피해·은행 분담 리스크 ↓, 단순 문의 셀프서비스화, 고령 고객 이탈 방지, 지역 포용금융(ESG) 실현 |

## 2. 시스템 구성도 (Architecture)

```
[사용자 입력]
  ① 대화(음성/텍스트)        ② 거래 이벤트(이체/출금/만기)   ← 은행 고유 데이터
        │                              │
        ▼                              ▼
  [의도 라우터]                  [거래 모니터]
  asset/fraud/explain            이상 룰 매칭 → 트리거
        │                              │
        └───────────┬──────────────────┘
                    ▼
        [AI Agent 오케스트레이터]  plan → tool → self-check → 응답 (선제 모드 지원)
        │           │            │              │
        ▼           ▼            ▼              ▼
   [RAG 검색]   [FDS 사기탐지]  [쉬운말 생성]   [액션/연계]
   BM25+Dense   룰 엔진+         LLM            알림/상담사
   +RRF+Rerank  위험 스코어러                   핸드오프
   (근거 인용)   →신호등🟢🟡🔴
                    │
                    ▼
              [가드레일] 근거 없음→"사람 연결" / 사기→"송금 멈춤"
                    │
                    ▼
   [프론트엔드/UI]  Streamlit — 음성 + 큰 글씨·고대비 카드 + 원터치 액션
```
**구성요소 역할 요약**
- **AI Agent/모델**: 의도 파악·도구 오케스트레이션·선제 트리거 (EXAONE-3.5 / HyperCLOVA X SEED)
- **검색·RAG**: 금융 문서 근거 검색 + 인용 답변 (BM25+BGE-M3+Qdrant+RRF+Reranker)
- **FDS**: 거래 이벤트 이상 룰 판정 → 위험 신호등
- **DB/외부연동**: Qdrant(벡터), 거래 이벤트 스트림(데모는 합성), (확장) 통신사 보이스피싱 신호
- **관리자 뷰(F6)**: JB 본부 상황판 — 차단 결과를 집계해 ₩막은 송금액·전북전남 보호 현황·차단 로그·ROI로 시각화
- **프론트엔드/UI**: 고령 친화(음성·큰 글씨·고대비·원터치). ※ **제출 데모는 컨셉 목업(웹 HTML/JS)**, 본 구현은 Streamlit + FastAPI(서빙)로 계획

## 3. 핵심 기능 명세 (Feature Specification)

| 기능명 | 기능 설명 | 입력 → 출력 데이터 | 관련 기술·알고리즘 | 구현 여부 |
|---|---|---|---|---|
| **F2. 거래단 사기 방어 「송금 직전 30초」 ★히어로** | **2층 구조** — (1층)FDS 가이드라인 룰 + (2층)**사용자별 평소 거래 베이스라인 대비 이탈** 탐지 → 송금 직전 선제 차단. 같은 금액도 평소 패턴 유무로 통과/차단 구분. **상시 감시 모드**: 24시간 거래 스트림 자동 모니터링·자동 차단 | 거래 이벤트 → 위험 신호등🟢🟡🔴+권고 조치 | FDS 룰 + 개인화 베이스라인(z-score·신규수취인·시간대 이탈), 가중 위험 스코어러 | **O** |
| **F1. 생애주기 자산 케어** | 연금·예금만기·의료비·상속을 고령 눈높이로 안내·요약, 근거 조항 인용 | 프로필·자연어 질문 → 쉬운말 답변+근거 출처 | 하이브리드 RAG(BM25+BGE-M3+RRF+Reranker), LLM | **O** |
| **F3. 쉬운말 금융비서** | "이번 달 왜 돈이 부족해?" 거래내역 해설·전문용어 풀이 + 음성 읽어주기·말로 질문 | 자연어 질문 → 음성+카드 응답 | STT/TTS, RAG, LLM | △(부분) |
| **F4. 선제 알림 & 사람 연결** | 만기·의료비·이상거래를 먼저 알림, 상담사/가족 핸드오프 | 이벤트 → 알림+원터치 액션 | 이벤트 트리거, 에이전트 플래너 | △(데모 목업·실구현 설계) |
| **F5. 가드레일** | 근거 없으면 단정 금지·"상담사 연결", 사기 의심 시 "송금 멈춤" 우선 | 응답 후보 → 검증된 응답 | 규칙 기반 후처리, 근거 검증 | **O** |
| **F6. JB 본부 상황판 (관리자 대시보드)** | 막은 송금액(₩)·전북전남 보호 현황·실시간 차단 로그·ROI를 JB 담당자 화면으로 제공 | 차단 이벤트 → ₩누적·지도·로그·FDS 분담 절감 추정 | 집계·시각화, 룰/베이스라인 결과 연동 | **O** |

## 4. 주요 기능 흐름도 (Flow)

**반응형 (어르신이 물을 때)**
```
대화 입력 → 의도 분류 → (fraud?) → 즉시 경고
                       → (asset/explain) → RAG 검색 → 근거 인용 답변
                       → 가드레일 검증 → 큰 글씨 카드 출력
```
**선제형 (거래가 발생할 때) — 핵심 차별점**
```
거래 이벤트 수신 → 이상 룰 평가 → 위험 점수 산출
   → (안전🟢) 통과 / (주의🟡·위험🔴) "먼저" 경고 카드 + 원터치 액션(상담사/가족/112)
```

## 5. 향후 발전 방향 (Future Work)

- **사기탐지 고도화**: 룰 → GNN 기반 이상거래 탐지(계좌 관계·사기링 포착), 연합학습(프라이버시 보존)
- **모델 고도화**: 한국 금융 도메인 sLLM 파인튜닝, HyperCLOVA X 기반 그룹 온프레미스 배포
- **음성 강화**: STT/TTS 고령 음성 최적화, 통신사 안티딥보이스 신호 연계
- **확장**: 가족 연계 보호 → 소상공인 사장님 자금 케어 → 전 연령 생애주기 케어
- **운영 고려**: 개인정보보호(최소수집·온프레미스), 금융소비자보호법 준수, 상담사 휴먼인더루프
